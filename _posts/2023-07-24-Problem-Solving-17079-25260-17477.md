E_j$$의 조건을 만족해야 한다. 즉, 이벤트가 종료될 때까지 있어야 하지만, 중간에 이벤트를 참여하는 것은 가능하다. 아래 쿼리 $$Q$$개를 처리하는 프로그램을 작성하라.    - 질의: $$s$$번 이벤트로부터 시작해 $$e$$번 이벤트에서 마무리하고자 할 때, 최대한 적은 개수의 이벤트를 거치고 싶다. 가능한 최솟값을 출력하라. 만약, 가능한 경우가 없다면 `impossible`을 출력하라.- 제약 조건    - $$1≤N≤100000$$이다.    - $$1≤Q≤100000$$이다.    - 모든 $$1≤i≤N$$에 대해 $$1≤S_i<E_i≤10^9$$을 만족한다.    - 모든 쿼리에서 $$1≤s≤N, 1≤e≤N$$이다. $$s≠e$$임은 보장되지 않음에 유의하라.- 서브테스크    - 서브테스크 1. 모든 이벤트 $$i$$에 대해 $$i$$에서 $$j$$번 이벤트로 넘어갈 수 있는 $$j$$는 정확히 하나이다.    - 서브테스크 2. $$N≤1000, Q≤100$$이다.    - 서브테스크 3. $$N≤5000$$이다.    - 서브테스크 4. $$Q≤100$$이다.    - 서브테스크 5. $$S_i≤S_j<E_j≤E_i$$인 두 $$i$$와 $$j$$는 존재하지 않는다.    - 서브테스크 6. 추가 제약 조건 없음## 서브테스크 5.$$S_i ≤S_j < E_j ≤ E_i$$인 $$i$$와 $$j$$가 존재하지 않으므로, $$E_i$$가 증가하는 순으로 정렬했을 때, $$S_i$$ 역시 증가한다. 고로, $$E_i$$가 증가하는 순으로 정렬된 상태에서 $$i$$번 이벤트에서 갈 수 있는 이벤트 번호는 $$[i + 1, f(i)]$$로 표현됨을 알 수 있으며, $$f(i)$$는 이분탐색을 이용해 $$O(\log N)$$의 시간에 구할 수 있다.따라서 $$i→f(i)$$로 가는 간선으로 이루어진 `sparse graph`에서 $$s$$에서 $$e$$ 이상인 정점으로 도달하기 위한 최소 이동 횟수가 답이 되므로 `sparse table`을 이용해 쿼리당 $$O(\log N)$$에 답을 구할 수 있다.전체 시간복잡도는 $$O((N+Q)\log N)$$이다.## 만점 풀이서브테스크 5의 아이디어를 확장하면 만점 풀이를 얻을 수 있다. 하지만 서브테스크 5와 같은 접근 방식으로 접근할 경우, 이벤트 번호가 연속된 하나의 구간으로 표현되지 않는다. 즉, 가능한 가장 큰 번호 $$f(i)$$를 저장하는 방식으로는 문제를 해결할 수 없다. 만약 $$E_{f(i)}$$가 $$E_e$$보다 크면 그 이벤트로 가면 안되고, $$E_e$$보다 끝나는 시각이 작은 이벤트 중 가장 이벤트 번호가 큰 이벤트로 가야 하기 때문이다. 즉, $$e$$에 따라 가야 하는 이벤트가 달라지기 때문에 $$f(i)$$를 저장한 `sparse table`로 문제를 해결할 수 없다.하지만, 놀랍게도 **반대로 정의하는 경우에는 가능**하다(이 부분을 생각하지 못해 솔브하지 못했다). $$i$$번 이벤트로 올 수 있는 이벤트들 중 가장 작은 이벤트 번호 $$g(i)$$를 저장한 `sparse table`을 생각하자. 자명히 $$S_{g(i)}$$가 $$S_s$$보다 크다면 $$g(i)$$에서 $$i$$로 오는 것이 최적이다. 만약 그렇지 않고 $$S_{g(i)}≤S_s$$라면, 아래 두 가지 경우가 가능하다.- $$E_s \geq S_i$$인 경우: $$s$$에서 $$i$$로 갈 수 있으므로, $$i$$까지 이동한 횟수에 $$1$$을 더한 값이 답이다.- $$E_s < S_i$$인 경우: $$s$$에서 $$i$$로 바로 갈 수 없으므로, 적어도 두 번의 이동이 필요하다. 그런데, $$g(i)$$에서 $$i$$로 갈 수 있으므로 $$E_s < S_i ≤ E_{g(i)}$$가 되어 $$s$$에서 $$g(i)$$로 이동할 수 있다($$E_s>S_s≥S_{g(i)}$$). 따라서 두 번만에 $$s$$에서 $$i$$로 가는 실례가 존재하므로 $$i$$까지 이동한 횟수에 $$2$$을 더한 값이 답이다.즉, $$g(i)$$를 저장하는 `sparse table`로 문제를 해결할 수 있다. $$f(i)$$를 저장하는 경우 해결할 수 없었던 이유는 다음 이벤트로 넘어갈 수 있는 조건이 대칭적이지 않은 $$S_i ≤E_j ≤E_i$$여서 이미 $$E_i>E_e$$인 $$i$$로 가는 순간 절대로 $$e$$로 갈 수 없기 때문이다.설명은 정렬된 배열을 가정하고 하였으나, 실제로 정렬이 필수적으로 요구되지는 않는다. $$k=g(i)$$의 정의를 $$k$$번 이벤트에서 $$i$$번 이벤트로 갈 수 있는 $$k$$ 중 $$S_k$$가 가장 작은 $$k$$로 바꾸게 되면 정렬 없이도 같은 정의를 할 수 있다.이제 $$k$$를 빠르게 구해보자. $$k$$번 이벤트에서 $$i$$번 이벤트로 갈 수 있을 조건이 $$S_i ≤E_k ≤E_i$$이므로 $$E_k$$의 인덱스에 $$S_k$$의 값과 최솟값의 인덱스 $$k$$를 관리하는 세그먼트 트리를 형성하면, 구간 $$[S_i, E_i]$$에서 최솟값에 해당하는 인덱스 $$k$$를 $$O(\log n)$$의 시간에 구할 수 있다. 문제는 최솟값에 쿼리에 의해 $$A_i$$가 변경될 때마다 $$B_i$$는 $$1$$ 증가한다.- 제약 조건    - $$1≤N≤500000$$이다.    - $$-10^9 ≤ A_i ≤ 10^9$$이다.    - 쿼리의 개수 $$M$$은 $$1≤M≤500000$$이다.    - $$1≤L≤R≤N$$과 $$-2000 ≤ X ≤ 2000$$, $$-10^9 ≤ Y ≤ 10^9$$을 만족한다.    - 4번 쿼리는 한 번 이상 주어진다.- **노트: segment tree beats 연습문제임**## 만점 풀이처음 잡은 `return condition`과 `direct update condition`은 아래와 같았다.- MAX 업데이트    - `return condition`: 구간이 겹치지 않거나, 최솟값이 $$Y$$ 이상인 경우    - `direct update condition`: 구간이 겹치고, 최댓값이 $$Y$$ 미만인 경우- MIN 업데이트    - `return condition`: 구간이 겹치지 않거나, 최댓값이 $$Y$$ 이하인 경우    - `direct update condition`: 구간이 겹치고, 최솟값이 $$Y$$ 초과인 경우이렇게 잡으면, 퍼텐셜 함수 $$f([s, e])$$를 구간 $$[s, e]$$에 있는 서로 다른 수의 개수라고 정의했을 때, 항상 재귀 호출에서 퍼텐셜 함수가 감소할 것으로 예상했으나, 이렇게 되면 값이 $$Y$$를 기준으로 하여 큰 수와 작은 수가 모두 있을 때 재귀 호출이 일어나고, 만약 $$Y$$보다 큰 수가 모두 같고, $$Y$$보다 작은 수가 모두 같다면 재귀 호출이 계속 일어나도 퍼텐셜 함수가 감소하지 않기 때문에 TLE를 받는다.이를 해결하려면 [수열과 쿼리 26](https://www.acmicpc.net/problem/17474)에서 사용한 아이디어인 **strict second maximum**의 아이디어를 차용해야만 한다. 그러면 아래와 같이 `return condition`과 `direct update condition`이 바뀐다.- MAX 업데이트    - `return condition`: 구간이 겹치지 않거나, 최솟값이 $$Y$$ 이상인 경우    - `direct update condition`: 구간이 겹치고, 최솟값은 $$Y$$보다 작으나 두 번째로 작은 값이 $$Y$$ 이상인 경우- MIN 업데이트    - `return condition`: 구간이 겹치지 않거나, 최댓값이 $$Y$$ 이하인 경우    - `direct update condition`: 구간이 겹치고, 최대값은 $$Y$$보다 크나 두 번째로 큰 값이 $$Y$$ 이하인 경우여기서 마지막으로 생각해야 하는 부분은 lazy 전파 조건이다. max 업데이트를 예로 들어 설명하면, 자식 노드도 최솟값이 $$Y$$보다 작아야 max 업데이트에 의한 변화가 일어나므로 그 경우에만 lazy를 전파해야 한다. 이 때, 이후에 일어난 다른 업데이트의 처리가 문제가 되는데, 두 노드가 propagation 함수가 호출되기 전까지 동일한 업데이트에 노출된다는 것을 이용하면 아래의 순서로 propagation을 진행하면 됨을 알 수 있다.- $$A_i$$ 변경 업데이트    - add 업데이트: 전부 전파    - max 업데이트: 자식 노드의 최솟값이 부모 노드의 최솟값 미만일 때 전파    - min 업데이트: 자식 노드의 최댓값이 부모 노드의 최댓값 초과일 때 전파        - min, max 업데이트는 add 업데이트가 적용된 후에 진행해야 한다. 부모 노드에서 전파되지 않은 add 업데이트가 있으면, 같은 add를 적용받지 않았기 때문에 비교가 안된다.- $$B_i$$ 변경 업데이트    - add 업데이트: 전부 전파    - max 업데이트: 자식 노드의 최솟값이 부모 노드의 최솟값과 같을 때 전파    - min 업데이트: 자식 노드의 최댓값이 부모 노드의 최댓값과 같을 때 전파        - $$A_i$$ 업데이트로 값이 바뀌었으므로, 위에서 max(min) 업데이트에 해당되었전 자식은 이제 최솟값(최댓값)이 같을 것이다. $$A_i$$ 업데이트와 같이 진행할 경우, 전파 조건이 동일하다.따라서 $$B_i$$의 변경을 위해 3종류의 lazy가 필요하므로 **최댓값, 최솟값, 두 번째로 큰 값, 두 번째로 작은 값, 최댓값의 개수, 최솟값의 개수, Add의 lazy, B 배열의 합, B 배열의 lazy(add, max, min)**의 총 **11개의 값(?!!)**을 관리하는 세그먼트 트리를 구현해야 한다. 아래는 구현 코드이다.```cpp#include <bits/stdc++.h>using namespace std;typedef long long ll;int n, q, arr[505050];const ll MIN = -1000000000000000000, MAX = 1000000000000000000;struct node {    ll maxVal, minVal, smaxVal, sminVal, maxCnt, minCnt;    ll addL;    ll valB;    ll addB, maxB, minB;} tree[2020202];node merger(node lt_val, node rt_val) {    node ret;    ret.maxVal = max(lt_val.maxVal, rt_val.maxVal);    if (lt_val.maxVal > rt_val.maxVal) {        ret.smaxVal = max(lt_val.smaxVal, rt_val.maxVal);        ret.maxCnt = lt_val.maxCnt;    }    if (lt_val.maxVal < rt_val.maxVal) {        ret.smaxVal = max(lt_val.maxVal, rt_val.smaxVal);        ret.maxCnt = rt_val.maxCnt;    }    if (lt_val.maxVal == rt_val.maxVal) {        ret.smaxVal = max(lt_val.smaxVal, rt_val.smaxVal);        ret.maxCnt = lt_val.maxCnt + rt_val.maxCnt;    }    ret.minVal = min(lt_val.minVal, rt_val.minVal);    if (lt_val.minVal < rt_val.minVal) {        ret.sminVal = min(lt_val.sminVal, rt_val.minVal);        ret.minCnt = lt_val.minCnt;    }    if (lt_val.minVal > rt_val.minVal) {        ret.sminVal = min(lt_val.minVal, rt_val.sminVal);        ret.minCnt = rt_val.minCnt;    }    if (lt_val.minVal == rt_val.minVal) {        ret.sminVal = min(lt_val.sminVal, rt_val.sminVal);        ret.minCnt = lt_val.minCnt + rt_val.minCnt;    }    ret.addL = 0;    ret.valB = lt_val.valB + rt_val.valB;    ret.addB = 0;    ret.maxB = 0;    ret.minB = 0;    return ret;}void addDirect(int v, int st, int ed, ll x) {    tree[v].addL += x;    tree[v].maxVal += x;    tree[v].minVal += x;    if (tree[v].smaxVal != MIN) tree[v].smaxVal += x;    if (tree[v].sminVal != MAX) tree[v].sminVal += x;}void maxDirect(int v, int st, int ed, ll x) {    tree[v].minVal = x;    tree[v].maxVal = max(tree[v].maxVal, x);    if (tree[v].minVal == tree[v].maxVal) {        tree[v].smaxVal = MIN;        tree[v].sminVal = MAX;        tree[v].maxCnt = ed - st + 1;        tree[v].minCnt = ed - st + 1;    }    else tree[v].smaxVal = max(tree[v].smaxVal, x);}void minDirect(int v, int st, int ed, ll x) {    tree[v].maxVal = x;    tree[v].minVal = min(tree[v].minVal, x);    if (tree[v].maxVal == tree[v].minVal) {        tree[v].smaxVal = MIN;        tree[v].sminVal = MAX;        tree[v].maxCnt = ed - st + 1;        tree[v].minCnt = ed - st + 1;    }    else tree[v].sminVal = min(tree[v].sminVal, x);}void valChange(int v, int st, int ed, int p) {    tree[v].valB += (ll)(ed - st + 1) * tree[p].addB;    tree[v].addB += tree[p].addB;    if (!p || tree[v].minVal == tree[p].minVal) {        tree[v].valB += tree[v].minCnt * tree[p].minB;        tree[v].minB += tree[p].minB;    }    if (!p || tree[v].maxVal == tree[p].maxVal) {        tree[v].valB += tree[v].maxCnt * tree[p].maxB;        tree[v].maxB += tree[p].maxB;    }}void propagation(int v, int st, int ed) {    if (st == ed) return;    int mid = (st + ed) / 2;    if (tree[v].addL) {        addDirect(2 * v, st, mid, tree[v].addL);        addDirect(2 * v + 1, mid + 1, ed, tree[v].addL);        tree[v].addL = 0;    }    if (tree[2 * v].maxVal > tree[v].maxVal)        minDirect(2 * v, st, mid, tree[v].maxVal);    if (tree[2 * v + 1].maxVal > tree[v].maxVal)        minDirect(2 * v + 1, mid + 1, ed, tree[v].maxVal);    if (tree[2 * v].minVal < tree[v].minVal)        maxDirect(2 * v, st, mid, tree[v].minVal);    if (tree[2 * v + 1].minVal < tree[v].minVal)        maxDirect(2 * v + 1, mid + 1, ed, tree[v].minVal);    if (tree[v].addB != 0 || tree[v].minB != 0 || tree[v].maxB != 0) {        valChange(2 * v, st, mid, v);        valChange(2 * v + 1, mid + 1, ed, v);        tree[v].addB = 0;        tree[v].minB = 0;        tree[v].maxB = 0;    }}void init(int v, int st, int ed) {    if (st == ed) {        tree[v].maxVal = arr[st];        tree[v].minVal = arr[st];        tree[v].smaxVal = MIN;        tree[v].sminVal = MAX;        tree[v].maxCnt = 1;        tree[v].minCnt = 1;        tree[v].addL = 0;        tree[v].valB = 0;        tree[v].addB = 0;        tree[v].maxB = 0;        tree[v].minB = 0;        return;    }    int mid = (st + ed) / 2;    init(2 * v, st, mid);    init(2 * v + 1, mid + 1, ed);    tree[v] = merger(tree[2 * v], tree[2 * v + 1]);}void addUpdate(int v, int st, int ed, int lt, int rt, ll x) {    if (lt > ed || rt < st) return;    if (lt <= st && ed <= rt) {        addDirect(v, st, ed, x);        tree[v].valB += (ed - st + 1);        tree[v].addB += 1;        return;    }    propagation(v, st, ed);    int mid = (st + ed) / 2;    addUpdate(2 * v, st, mid, lt, rt, x);    addUpdate(2 * v + 1, mid + 1, ed, lt, rt, x);    tree[v] = merger(tree[2 * v], tree[2 * v + 1]);}void maxUpdate(int v, int st, int ed, int lt, int rt, ll x) {    if (lt > ed || rt < st || tree[v].minVal >= x) return;    if (lt <= st && ed <= rt && tree[v].sminVal > x) {        maxDirect(v, st, ed, x);        tree[v].valB += tree[v].minCnt;        tree[v].minB += 1;        return;    }    propagation(v, st, ed);    int mid = (st + ed) / 2;    maxUpdate(2 * v, st, mid, lt, rt, x);    maxUpdate(2 * v + 1, mid + 1, ed, lt, rt, x);    tree[v] = merger(tree[2 * v], tree[2 * v + 1]);}void minUpdate(int v, int st, int ed, int lt, int rt, ll x) {    if (lt > ed || rt < st || tree[v].maxVal <= x) return;    if (lt <= st && ed <= rt && tree[v].smaxVal < x) {        minDirect(v, st, ed, x);        tree[v].valB += tree[v].maxCnt;        tree[v].maxB += 1;        return;    }    propagation(v, st, ed);    int mid = (st + ed) / 2;    minUpdate(2 * v, st, mid, lt, rt, x);    minUpdate(2 * v + 1, mid + 1, ed, lt, rt, x);    tree[v] = merger(tree[2 * v], tree[2 * v + 1]);}ll get(int v, int st, int ed, int lt, int rt){    propagation(v, st, ed);    if (lt > ed || rt < st) return 0;    if (lt <= st && ed <= rt) return tree[v].valB;    int mid = (st + ed) / 2;    return get(2 * v, st, mid, lt, rt) + get(2 * v + 1, mid + 1, ed, lt, rt);}int main() {    ios_base::sync_with_stdio(false);    cin.tie(0);    cin >> n;    for (int i = 1; i <= n; i++) cin >> arr[i];    init(1, 1, n);    cin >> q;    while (q--) {        int t, l, r; ll x;        cin >> t;        if (t == 1) {            cin >> l >> r >> x;            if (x != 0) addUpdate(1, 1, n, l, r, x);        }        if (t == 2) {            cin >> l >> r >> x;            maxUpdate(1, 1, n, l, r, x);        }        if (t == 3) {            cin >> l >> r >> x;            minUpdate(1, 1, n, l, r, x);        }        if (t == 4) {            cin >> l >> r;            cout << get(1, 1, n, l, r) << "\n";        }    }    return 0;}```### 추가사항이번에 코드를 구현할 때, 이전과 다른 방식의 코드를 기반으로 구현하였다. 아래는 기존 코드와의 차이점을 명시한 새로운 기반 코드이다.```cpp#include <bits/stdc++.h>using namespace std;struct seg {    struct node {	int val, lazy;    } tree[/*size*/];    void direct_update(int v, ...) {	// change v value and lazy    }    void propagation(int v, int st, int ed) {        //propagate update to child node: lazy propagation and value change    }    node merge_operation(node lt_val, node rt_val) {        return /*result*/;    }    void init(int v, int st, int ed) {        if (st == ed){            tree[v] = /*initial value*/;            return;        }        int mid = (st + ed) / 2;        init(2 * v, st, mid);        init(2 * v + 1, mid + 1, ed);	tree[v] = merge_operation(tree[2 * v], tree[2 * v + 1])    }    void update(int v, int st, int ed, int lt, int rt, int x) {        if (/*return condition*/) return;        if (/*direct update condition*/) {            direct_update(v, ...)            return;        }        propagation(v, st, ed);        int mid = (st + ed) / 2;        update(2 * v, st, mid, lt, rt, x);        update(2 * v + 1, mid + 1, ed, lt, rt, x);        tree[v] = merge_operation(tree[2 * v], tree[2 * v + 1]);    }    node get(int v, int st, int ed, int lt, int rt) {        propagation(v, st, ed);        if (lt > ed || st > rt) return /*zero-value*/;        if (lt <= st && ed <= rt) return tree[v];        int mid = (st + ed) / 2;        return merge_operation(            get(2 * v, st, mid, lt, rt),            get(2 * v + 1, mid + 1, ed, lt, rt)        );    }};```기존 코드와의 차이점은 다음과 같다(강조표시):- `propagation(v, st, ed)`: update를 **자식 노드에 전파하고, 자식 노드의 값을 바꾸는** 과정이다. 즉, 어떤 정점 `v`의 `lazy[v]`는 이미 그 lazy에 대한 업데이트는 끝났고, 그 lazy가 전파되지 않았다는 뜻이다. **위치가 변경**되었음에 유의하라.- `**return condition**`: 굳이 update를 전파할 필요가 없는 조건이다.- `**direct update condition**`: 바로 update를 하는 조건이다. 내부 구현도 달라졌는데, 기존에는 lazy 값을 변경시킨 후 `propagation(v, st, ed)`를 호출했다면, **지금은 `direct_update(v, ...)` 함수를 이용해 직접 값과 lazy를 변경**시킨다. 이 `direct_update(v, ...)` 함수는 `propagation(v, st, ed)`에서 직접적으로 사용될 수 있다.- `merge_operation(lt_val, rt_val)`: 자식 노드의 두 값을 이용해 부모 노드의 값을 구하는 연산이다.- `zero-value`: 구간에 포함되지 않을 경우 리턴하는 값이다. 일반적인 구간 합 구하기에서는 `0`을 리턴한다.위 방법으로 구현할 경우, 부모 노드와 자식 노드 사이의 관계를 이용한 lazy 전파가 가능하다. 이 문제에서는 **부모 노드와 자식 노드의 최댓값 및 최솟값이 같은지 여부**에 따라 lazy 전파가 일어나기 때문에 기존 방식으로 구현하기에는 한계가 있다.
